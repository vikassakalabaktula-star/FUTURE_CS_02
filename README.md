# Phishing Detection & Awareness Project

A cyber security internship project analyzing simulated phishing emails to identify common red flags and build a non-technical awareness guide for employees.

## 📌 Overview

This project reviews two simulated phishing emails impersonating a physical security vendor and an internal HR/compliance team. Each email is broken down into plain-language indicators so that both technical and non-technical readers can learn to recognize similar attacks.

## 🛠️ Tools Used

- Manual email header inspection (sender domain vs. display name)
- Manual link/URL verification
- Content and tone analysis (urgency, fear, generic greetings)

## 🔍 Analysis Approach

1. Collected two phishing email samples
2. Inspected sender display name vs. actual sending domain
3. Identified suspicious links and call-to-action patterns
4. Flagged social engineering techniques (urgency, fear, curiosity)
5. Classified each email's risk level (Safe / Suspicious / Phishing)
6. Documented findings in a structured report
7. Compiled prevention and awareness guidelines for employees

## ✅ Key Findings

Both emails were sent from the same lookalike domain while impersonating two unrelated organizations, and both relied on urgency (curiosity in one case, fear of punishment in the other) to pressure the reader into clicking a link.

## 🛡️ Awareness Takeaways

- Always check the sender's actual email domain, not just the display name
- Be cautious of any message creating urgency or fear
- Verify unexpected requests through a separate, known channel
- Never enter credentials after clicking a link from an email
- Report suspicious emails to IT/Security rather than replying
